# b(ru)h-(st)

This is where I'm exploring what makes Rust different and "the modern systems programming" language. My goal is to build a distributed file system or database engine in Rust because I won't be on campus in the fall to build it in C, and I'd love to explore this language more for possible low-level applications like lock-free libraries and RTOS usage.

- [ ] recreate some miniutils (most likely tools like grepit, timeit, nmapit, and the like)
- [ ] build a simple shell/process scheduler

So far, I've built a guessing game with standard I/O and unsigned 32-bit integers... but even Usain Bolt was once a baby.
